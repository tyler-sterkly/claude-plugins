# ext-website

Generates a complete, hosting-ready frontend website for a Firefox browser extension. Writes all files directly to `.website/` inside the extension project directory.

## When to trigger

- "generate a website"
- "build a website"
- "create a landing page"
- "build an extension site"
- "make me a website for my extension"

## How it works

1. **Gather info** — asks for extension directory, name, domain, www preference, tracking script URL, AMO slug, extension ID, which nav pages to include, and whether to add a cookie consent banner.
2. **Read extension files** — reads `.docs/LISTING.txt`, `CLAUDE.md`, `README.md`, and `manifest.json` to generate copy, FAQ, features, and about content from what the extension actually does.
3. **Generate color scheme and typography** — produces a full CSS custom property palette based on the extension name and purpose. All colors and fonts live in one block at the top of `css/style.css`.
4. **Resolve assets** — finds logo, icons, and favicon in the extension directory and copies them to `.website/assets/`.
5. **Write all pages** — generates every page file listed below.
6. **Report** — summarizes what was generated, assets found, color scheme, font choices, and any fallbacks used.

## Inputs

| Field | Required | Notes |
|---|---|---|
| Extension project directory | Yes | Scanned for copy, icons, and config |
| Extension name | Yes | Used in copy, nav, and footer |
| Website domain | Yes | Determines FormSubmit target and contact email |
| Use www prefix | Yes | |
| Tracking script URL | Yes | Placed on index.html only |
| AMO slug | Yes | Used in tracking script and install button URL |
| Extension ID | Yes | Used in tracking script |
| Nav pages | Yes | Choose from: About, FAQ, Features |
| Cookie consent banner | Yes | yes or no |

## Output — pages written to `.website/`

| Page | Notes |
|---|---|
| `index.html` | Hero, feature highlights, install button, tracking script |
| `thanks/index.html` | Post-install confirmation, noindex, postMessage handler |
| `privacy/index.html` | Generated by `ext-legal` |
| `terms/index.html` | Generated by `ext-legal` |
| `contact/index.html` | FormSubmit form, success popup via JS |
| `uninstall/index.html` | Firefox/Chrome/Edge toggle, Windows/Mac sub-toggle, noindex |
| `about/index.html` | Optional — extension story and purpose |
| `faq/index.html` | Optional — accordion FAQ, 6-10 questions |
| `features/index.html` | Optional — feature list with visual treatment |
| `css/style.css` | Shared styles, full CSS custom property palette |
| `js/main.js` | Shared JS: nav, cookie consent, contact form, postMessage |

## Edge cases

- **No logo found** — falls back to the largest available icon PNG
- **No .docs/LISTING.txt** — derives copy from README.md and manifest.json description
- **privacy and terms pages** — handled entirely by `ext-legal`; do not write placeholder content for these
- **Tracking script** — placed on `index.html` only, never on any other page
- **Install button** — placed on `index.html` only, never on subpages

## Related skills

- `ext-legal` — called automatically to generate privacy and terms pages
- `ext-icons` — produces the logo.png and icons this skill looks for
